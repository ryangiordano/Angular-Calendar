Thank you for taking a look at my code.

I appreciated the challenge and did my best to satisfy the requirements.

Features

I was able to implement the following features:

  • Generate a list of calendars weekdays in green, weekends in yellow, selected days in orange, and holidays in purple.
  • Selected days spill over into the next month or next year.
  • Days before and after on the calendar that do not belong to the given month are in grey.
  • Hover over the days and you can see the name of the holiday if it applies. If not, you can still see the exact day.
  • Though it was not explicitly stated in the guidelines, I assumed Country Code was for getting the holidays for a specific country. You can see holidays from many different countries.

Bugs

There were several bugs that I was not able to address along the way.

  • Country codes JA and CN (Japan and China) cause the Holiday plugin I was using to crash. Tested: FR, CA, CU, US.
  • There appears to be a deprecated component to the library. In production, I would perhaps use a paid API.

Would've, Could've, Should've

Things I would have liked to add:

  • Cleaner interface with transitions for the calendars on load.
  • Refactoring algorithm to generate calendars. There are some silly patches I applied to code that could stand to be polished.
  • Error handling.
  • Two-way data-binding for the calendars so that the rendered calendars updated dynamically as the values in the forms changed.
  • Stronger validation for the form.
